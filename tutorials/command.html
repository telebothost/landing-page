Adding and Configuring Commands

An expanded, easy-to-follow step-by-step guide for first-time users. Each step tells you exactly what to click and what to type.

Quick recap (if you followed Tutorial 2)

You already added your bot and created a simple /start command. This page shows more steps and explains each option clearly so you can build useful commands.

Overview — What is a Command?

A command is a trigger (like /start or hi) that tells your bot what to do. When a user sends that trigger, your bot will respond with the message or action you set.

Step 1 — Open the Commands tab

  1. Go to your TeleBot Host dashboard and select the bot you want to manage.
  2. Click the Commands tab in the bot panel. You will see a list of existing commands.
  3. If this is empty, don’t worry — we will add commands now.

Step 2 — Add a new command

  1. Click the + (Add Command) button.
  2. A form appears with fields. Fill them as shown below (we explain each one after this).
  3. When done, click Save. Your new command appears in the commands list.

Step 3 — What each field means (easy, one-by-one)

Command Name

This is the word users will type to trigger the command. Examples:

  • /start — common welcome command
  • /help — shows help text
  • hi — works without slash if you want
Tip: Keep it short and clear. Prefer slashes for main commands (for example /start).

Bot Response

Text the bot will send when the command is used. This can be a short message or a formatted message using Markdown. Example:

Welcome! Use these options:
- Info
- Support
If you add both a Bot Response and code, the response will be sent automatically while code can do more actions. Use either or both.

Keyboard

Optional buttons shown with the message. Use a comma (,) to separate buttons on the same row and \n to start a new row.

Same row example: Yes, No → shows Yes and No next to each other.
New row example: Option 1\nOption 2 → Option 1 on first row, Option 2 on second row.
If you choose a keyboard type that expects an answer, make sure your keyboard options match what you expect users to tap.

Need Reply

If you enable Need Reply, the bot will wait for the user's next message and treat it as the answer to this command. Use this for forms or questions.

Example: Command asks "Please enter your email" and you enable Need Reply. The next message the user sends will be captured as the email.
Important: Use Need Reply only when you actually want to collect user input. It creates a short session for that user.

Aliases

Other triggers that will run the same command. Add common variations so users can type different words and still get the same result.

Command: /hello
Aliases: hi, hey, hola
Add aliases as comma-separated words (no slashes needed). Aliases make your bot more forgiving to different user inputs.

Allow Only Group (deprecated)

This setting is old and not recommended. You can ignore it for new commands.

Step 4 — Save and test the command (easy test)

  1. After saving your command, open Telegram and go to your bot.
  2. Type the command exactly (for example /start or hi).
  3. Confirm the bot replies with the Bot Response and shows keyboard buttons if you added them.

Step 5 — Example: Create a simple help command (walkthrough)

  1. Click + to add a command.
  2. Command Name: /help
  3. Bot Response: 
    Welcome! Choose one:
- Info
- Contact
  4. Keyboard: Info, Contact
  5. Need Reply: Off
  6. Save. Test in Telegram by typing /help.

Step 6 — Example: Collect user input (Need Reply example)

  1. Create a command with:
    • Command Name: /email
    • Bot Response: Please enter your email address
    • Need Reply: On
  2. Save the command.
  3. User types /email. The bot asks "Please enter your email address".
  4. User replies with their email. Your bot panel will receive that reply as the user's response for this command (check logs or use code later to store it).
How to use the reply later: after you receive the reply, you can edit the command to add code that saves the reply to a database or variable (advanced step).

Step 7 — Edit or delete a command

  1. In Commands list, click the command name to open the edit form.
  2. Change any field (name, response, keyboard, aliases, need reply).
  3. Click Save to apply changes.
  4. To delete: click the three dot on top then click on Delete button (confirm when asked).

Step 8 — Command priority and matching (simple)

When the bot receives a message, it checks for matching commands. Exact matches and slash commands have priority. If no command matches, the fallback * command runs (if set).

If two commands could match the same text, prefer unique names or use aliases carefully to avoid conflicts.

Special Commands in TBL (Easy Guide)

Learn the special commands in TBL in a simple way. No coding knowledge needed.

Important Rules:
  • All normal commands run inside a function.
  • @ runs differently: it is not function-wrapped.
  • Top-level return does not work in @.
  • @ is synchronous, but you can define async functions inside it if needed.
  • Use @ for shared settings and variables that other commands can access.

@ — Startup and Configuration

Think of the @ command as your bot’s “setup area.” It runs automatically when the bot starts and prepares everything your bot will use later.

  • Main Purpose: Store shared values and settings that all commands can use, like admin IDs, feature flags, or default messages.
  • How it works: @ runs once when the bot starts. Because it is not inside a function, normal return or early-exit commands do not behave the same as in other commands.
  • Async tasks: @ is synchronous, which means it runs step by step. If you need to do tasks that take time (like loading a file or fetching a small setting), you must define a separate async function inside @ and then call it carefully.
  • Do not: Put any user interaction logic in @. Commands that wait for user input should go in normal commands.
  • Do: Only define settings, shared values, or helper functions that other commands will use.
  • Example use: You can store the bot’s admin list, default greetings, or feature switches so that any other command can access them easily.

! — Error Handling

Runs automatically if a command produces an error. Use it to log problems or notify users that something went wrong.

@@ — After Every Command

Runs after each command. Useful for logging, cleaning temporary data, or counting command uses.

* — Fallback

Runs if no other command matches. Use it to guide users, show a menu, or suggest the next step.

* is master Command it can handle any type of updates that telegram support

/channel_update — Channel Posts

Runs when the bot receives channel update like channel post .

/inline_query — Inline Mode

Runs when users call your bot inline. Use it to show quick results or options instantly.

Tip: Keep @ for setup, ! for errors, @@ for after-command tasks, * for unknown messages, and channel/inline commands for their specific events.
